CROSS-CUTTING PATTERNS (All Three)
================================================================================

1. BILINGUAL BY DEFAULT
   Every slide: Thai + English (never monolingual)
   Examples: "Oracle คืออะไร?" / "What is Oracle?"

2. MODULAR STRUCTURE
   Separate topic tracks (philosophy, setup, main)
   Allows mixing/reusing independently
   Can present any track standalone

3. TIMESTAMPED WHEN IMPORTANT
   HHMM_filename.md format
   Prevents same-day collisions
   Tracks iterative improvements

4. QR CODES INCLUDED
   Link to GitHub repo
   Link to live slides
   Social media links
   Example: phukhao has qr-github.png, qr-facebook.png

5. ACCESSIBLE & INCLUSIVE
   Color contrast (WCAG standard)
   TTS-ready (class="tts-text" in HTML)
   Large readable fonts
   Monospace for code blocks

6. REPOSITORY INTEGRATION
   Slides live in /slides/ directory
   Tracked in git (history preserved)
   Can link to specific commits
   Part of codebase history

7. COLOR SYSTEM
   Cyan (#00d9ff) - Headers
   Green (#00ff9f) - Highlights
   Yellow (#ffd700) - Emphasis
   Purple (#bf7fff) - Accent

RECOMMENDED APPROACH BY USE CASE
================================================================================

WORKSHOP / COURSE MATERIALS
  → Use Markdown Outline (Pattern 2)
  → Easy to update, version-control friendly
  → Example: claude-code-workshops SIIT slides

PUBLIC PRESENTATION / CONFERENCE TALK
  → Use Reveal.js (Pattern 1)
  → Professional polish, multimedia support
  → Example: phukhao-oracle intro slides

QUICK DEMO / ONE-TIME PRESENTATION
  → Use Script Generation (Pattern 3)
  → Fast iteration, source material exists
  → Example: Nat-s-Agents rapid slide generation

PRODUCTION QUALITY + TEAM COLLABORATION
  → Hybrid: Markdown → Reveal.js HTML
  → Write in markdown (easy to edit)
  → Convert to HTML (professional output)
  → Optional: Add audio narration later
